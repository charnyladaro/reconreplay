Detection

What ReconReplay detects

Each vulnerability type has a dedicated detection engine — not just string matching, but time-based, behavioral, and differential analysis.

Critical

SQL Injection

DB error patterns, time-based delay (>4s and >3× baseline), boolean differential detection.

Critical

OS Command Injection

Detects ping output, whoami, uid= in response body. Triggers on timeout >9s.

High

Cross-Site Scripting

6 encoding variants checked. Playwright dialog handler catches live alert() / confirm() XSS execution.

High

SSRF

Internal IP ranges, cloud metadata endpoints (169.254.169.254), localhost variations.

High

Path Traversal

Detects /etc/passwd content and Windows hosts file content in responses.

High

IDOR

Adjacent numeric ID substitution — different valid 200 responses indicate unauthorized access.

Medium

CRLF Injection

Canary header (X-Audit) injection via %0d%0a sequences, verified in response headers.

Medium

Open Redirect

3xx responses with Location pointing to the injected domain. Covers protocol-relative bypasses.

🔒

100% Local Storage

All captured sessions, requests, and findings are stored in local SQLite databases. No pentest data ever leaves your machine.

🛡️

Encrypted Auth Tokens

Session tokens are stored encrypted using OS-level DPAPI (Windows). Your credentials are never stored in plaintext.

⚡

Identity Auth Only

Supabase is used exclusively for user identity. Zero target data, session data, or findings are ever synced to the cloud.
